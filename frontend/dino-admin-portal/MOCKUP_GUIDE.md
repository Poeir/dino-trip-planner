# 🦖 Dino Trip Planner - Admin Portal Mockup Guide

## 📖 เกี่ยวกับระบบ

**Dino Trip Planner** เป็นระบบวางแผนการท่องเที่ยวอัจฉริยะที่ใช้ **LLM (Large Language Model)** และ **Chatbot** ในการช่วยผู้ใช้วางแผนทริปท่องเที่ยวที่เหมาะสมที่สุด

### 🎯 วิธีการทำงานของระบบ:

1. **ผู้ใช้สนทนากับ Chatbot** 🤖
   - บอกความต้องการ เช่น "อยากไปเที่ยวกรุงเทพ 3 วัน งบ 5,000 บาท"
   - ระบุความชอบ เช่น ชอบวัฒนธรรม, ชอบอาหาร, ไปกับครอบครัว

2. **LLM วิเคราะห์และประมวลผล** 🧠
   - ดึงข้อมูลจากฐานข้อมูล (Places, Activities, Events)
   - วิเคราะห์ความต้องการของผู้ใช้
   - คำนวณระยะทาง งบประมาณ และเวลา
   - สร้างโปรแกรมทริป (Itinerary) ที่เหมาะสม

3. **ระบบสร้างโปรแกรมทริป** 📋
   - จัดลำดับสถานที่และกิจกรรมตามความเหมาะสม
   - แนะนำอีเวนต์พิเศษ (ถ้ามี) ในช่วงเวลานั้น
   - คำนวณค่าใช้จ่ายโดยประมาณ

---

## 🗂️ โครงสร้างข้อมูลในระบบ

### 1. 📍 Places (สถานที่ท่องเที่ยว)

**ประกอบด้วย:**
- ชื่อสถานที่
- ประเภท (วัด, ร้านอาหาร, โรงแรม, พิพิธภัณฑ์, สวนสาธารณะ)
- พิกัด GPS (Latitude, Longitude)
- คะแนนรีวิว (Rating)
- ระดับราคา (Price Level: 0-4)

**LLM ใช้ข้อมูลนี้:**
- กรองสถานที่ตามประเภทที่ผู้ใช้สนใจ
- เช็คว่าสถานที่อยู่ในงบประมาณหรือไม่
- คำนวณระยะทางระหว่างสถานที่
- จัดอันดับความน่าสนใจตามคะแนนรีวิว

**ตัวอย่างการใช้งาน:**
- ผู้ใช้: "อยากไปวัดในกรุงเทพที่มีคะแนนสูง"
- LLM: กรองเฉพาะ type = "temple" และ rating >= 4.5

---

### 2. 🎯 Activities (กิจกรรม)

**ประกอบด้วย:**
- ชื่อกิจกรรม
- หมวดหมู่ (outdoor, food, culture, shopping, wellness, entertainment, sports, education)
- คำอธิบาย
- แท็ก (family-friendly, budget, instagram-worthy)
- เหมาะสำหรับ (เดี่ยว/คู่/ครอบครัว/กลุ่ม)

**LLM ใช้ข้อมูลนี้:**
- แนะนำกิจกรรมตามความสนใจ
- เลือกกิจกรรมที่เหมาะกับลักษณะการเดินทาง
- กรองตาม Tags ที่ผู้ใช้ต้องการ

**ตัวอย่างการใช้งาน:**
- ผู้ใช้: "อยากทำกิจกรรมกลางแจ้งที่เหมาะกับครอบครัว"
- LLM: เลือก category = "outdoor" AND suitableFor.family = true

---

### 3. 🎉 Events (อีเวนต์/เทศกาล)

**ประกอบด้วย:**
- ชื่อและประเภทอีเวนต์
- วันที่เริ่ม-สิ้นสุด
- สถานที่จัดงาน
- ค่าเข้างาน (ฟรี หรือ มีค่าใช้จ่าย)

**LLM ใช้ข้อมูลนี้:**
- เช็คว่ามีอีเวนต์พิเศษในช่วงที่ผู้ใช้เดินทางหรือไม่
- แนะนำอีเวนต์ที่น่าสนใจ
- คำนวณค่าใช้จ่ายเพิ่มเติม (ถ้ามี)

**ตัวอย่างการใช้งาน:**
- ผู้ใช้: "จะไปกรุงเทพวันที่ 15-16 พฤศจิกายน 2026"
- LLM: พบว่ามี "เทศกาลลอยกระทง" → แนะนำให้ไปร่วมงาน

---

### 4. 🗺️ Itineraries (โปรแกรมท่องเที่ยว)

**ประกอบด้วย:**
- ชื่อและธีมทริป
- จำนวนวัน
- จุดหมาย (Stops) พร้อมเวลาและกิจกรรม
- สร้างโดย LLM หรือ ผู้ใช้
- สถานะ (สาธารณะ/ส่วนตัว)

**LLM ใช้ข้อมูลนี้:**
- สร้างโปรแกรมทริปใหม่จากความต้องการผู้ใช้
- จัดลำดับสถานที่ตามความเหมาะสม
- คำนวณเวลาเดินทางระหว่างจุด
- ใช้โปรแกรมที่มีอยู่เป็นตัวอย่างแนะนำผู้อื่น

**ตัวอย่างการใช้งาน:**
- ผู้ใช้: "อยากไปเที่ยวกรุงเทพ 3 วัน งบ 5,000 บาท ไปกับครอบครัว"
- LLM: สร้างโปรแกรมที่มี:
  - วันที่ 1: วัดพระแก้ว → ชิมอาหาร → พิพิธภัณฑ์สยาม
  - วันที่ 2: ตลาดนัดจตุจักร → สวนลุมพินี
  - วันที่ 3: กิจกรรมที่เหลือ
  - รวมค่าใช้จ่าย: ~4,800 บาท

---

## 🔄 Flow การทำงานของระบบ

```
[ผู้ใช้] 
   ↓ พูดคุยกับ
[Chatbot] 
   ↓ ส่งความต้องการไปที่
[LLM] 
   ↓ ดึงข้อมูลจาก
[Admin Portal - ข้อมูล Places, Activities, Events]
   ↓ วิเคราะห์และสร้าง
[Itinerary - โปรแกรมทริป]
   ↓ แสดงผลให้
[ผู้ใช้] พร้อมแผนทริปที่เหมาะสม
```

## 🎨 หน้า Admin Portal ทั้งหมด

### 1. **Dashboard (แดชบอร์ด)**
- แสดงภาพรวมจำนวนข้อมูลทั้งหมด
- Quick Actions สำหรับเพิ่มข้อมูลแต่ละประเภท
- กิจกรรมล่าสุดในระบบ
- สถานะการทำงานของระบบ

### 2. **Places Page (จัดการสถานที่)**
- แสดงรายการสถานที่ในรูปแบบ Table
- ค้นหาและกรองตามประเภท
- ฟอร์มเพิ่มสถานที่ใหม่พร้อมคำอธิบาย
- ลบและแก้ไขสถานที่

### 3. **Activities Page (จัดการกิจกรรม)**
- แสดงรายการกิจกรรมในรูปแบบ Grid Cards
- ค้นหาและกรองตามหมวดหมู่
- ฟอร์มเพิ่มกิจกรรมใหม่พร้อมระบุ "เหมาะสำหรับ"
- ลบและแก้ไขกิจกรรม

### 4. **Events Page (จัดการอีเวนต์)**
- แสดงรายการอีเวนต์ในรูปแบบ List
- กรองตามสถานะ (กำลังจะมาถึง/กำลังจัด/จบแล้ว)
- ฟอร์มเพิ่มอีเวนต์พร้อมช่วงวันที่
- ลบและแก้ไขอีเวนต์

### 5. **Itineraries Page (จัดการโปรแกรมทริป)**
- แสดงรายการโปรแกรมทั้งหมด
- กรองตาม LLM/User และ สาธารณะ/ส่วนตัว
- ฟอร์มเพิ่มโปรแกรมใหม่
- เปลี่ยนสถานะ สาธารณะ/ส่วนตัว
- ลบและแก้ไขโปรแกรม

## 📝 คำอธิบายในแต่ละฟิลด์ของฟอร์ม

### Places (สถานที่)
- **ชื่อสถานที่**: ใช้แสดงและค้นหาสถานที่
- **ประเภท**: LLM ใช้กรองและแนะนำตามประเภท
- **Latitude/Longitude**: แสดงบนแผนที่และคำนวณระยะทาง
- **คะแนน**: LLM จัดอันดับความน่าสนใจ
- **ระดับราคา**: กรองตามงบประมาณผู้ใช้

### Activities (กิจกรรม)
- **ชื่อกิจกรรม**: LLM แนะนำผ่าน Chatbot
- **หมวดหมู่**: จัดหมวดหมู่ตามความสนใจ
- **คำอธิบาย**: LLM สร้างคำแนะนำละเอียด
- **แท็ก**: ช่วยค้นหาและแนะนำที่ตรงใจ
- **เหมาะสำหรับ**: กรองตามลักษณะการเดินทาง

### Events (อีเวนต์)
- **ชื่อและประเภท**: LLM แนะนำอีเวนต์พิเศษ
- **วันที่**: ตรวจสอบว่าทริปตรงกับช่วงอีเวนต์
- **สถานที่จัดงาน**: คำนวณระยะทาง
- **ค่าเข้างาน**: คำนวณงบประมาณรวม

### Itineraries (โปรแกรมทริป)
- **ชื่อโปรแกรม**: แสดงและบันทึกโปรแกรม
- **ธีม**: LLM เลือกสถานที่ตามธีม
- **จำนวนวัน**: คำนวณจำนวนกิจกรรม
- **สร้างโดย**: วิเคราะห์ประสิทธิภาพ LLM
- **สถานะ**: ใช้เป็นตัวอย่างสำหรับผู้อื่น

## 📌 สิ่งที่ต้องพัฒนาต่อ (เมื่อเชื่อม Backend จริง)

2. **LLM Integration**
   - เชื่อม LLM (OpenAI GPT, Claude, etc.)

   - Fine-tune Model ด้วยข้อมูลจริง

3. **Chatbot**
   - สร้าง Chat Interface
   - เชื่อม Chatbot กับ LLM
   - บันทึกประวัติการสนทนา

4. **Advanced Features**
   - Map Integration (Google Maps)
   - Route Optimization
   - Budget Calculator
   - Weather Integration
   - Real-time Updates
